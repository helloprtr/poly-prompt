# Code of Conduct

## Our standard

This project is committed to a respectful, constructive, and welcoming community.

Examples of behavior that help build that environment:

- showing empathy and patience toward other people
- giving actionable technical feedback without personal attacks
- respecting different experience levels, backgrounds, and communication styles
- assuming good intent while still raising concrete concerns clearly
- keeping discussions focused on the work, the user impact, and the evidence

Examples of unacceptable behavior:

- harassment, discrimination, or intimidation
- insulting or demeaning comments
- bad-faith trolling or deliberately disruptive behavior
- publishing private information without consent
- repeated attempts to bypass project safety rules after being asked to stop

## Scope

This Code of Conduct applies in issues, pull requests, discussions, release conversations, and other project communication channels.

## Enforcement

Project maintainers may edit, remove, or reject comments, issues, pull requests, or other contributions that violate this Code of Conduct.

## Reporting

For conduct concerns, contact the maintainers through a private GitHub channel when possible.

For security issues, follow [SECURITY.md](SECURITY.md) instead of this process.
