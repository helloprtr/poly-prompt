# Security Policy

## Supported Versions

The project is currently maintained on the latest public minor line.

| Version | Supported |
| --- | --- |
| `0.6.x` | Yes |
| `< 0.6.0` | No |

## Reporting a Vulnerability

Please use GitHub Private Vulnerability Reporting for this repository.

- Open a private report from the repository's Security tab
- Include the affected version, platform, and a minimal reproduction
- Share whether the issue affects local history, repo memory, launch handoff, clipboard handling, or release artifacts

Please do not open a public issue for a security vulnerability.

## What to Include

Useful reports usually include:

- the exact `prtr version`
- operating system and terminal app
- the command that triggered the issue
- whether the issue involved `open-copy`, `exec`, `server`, or release artifacts
- logs, screenshots, or reproduction steps with secrets removed

## Response Expectations

The goal is to:

- acknowledge valid reports quickly
- reproduce and assess severity
- ship a fix or mitigation
- publish a coordinated advisory when needed

There is no formal SLA yet, but actively maintained releases receive priority.
