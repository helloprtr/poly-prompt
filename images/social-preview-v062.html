From context switching to command flow

prtr is built for the messy part around LLM use: logs, app choice, repeated follow-ups, and project-specific language that should survive the next run. 

$ npm test 2>&1 | prtr go fix "왜 깨지는지 정확한 원인만 찾아줘"
$ prtr swap gemini
$ prtr take patch
$ prtr learn
