prtr watch — 자동 제안 UX

백그라운드에서 컨텍스트를 감지하고 제안할 때 어떤 방식이 맞아?

● npm test 2 failed
─────────────────────────
prtr 에러 감지됨 — AI에 보낼까? [y]/n

A. Inline 제안

에러/이벤트 직후 터미널에 한 줄 제안. y 누르면 즉시 전송.

● npm test 2 failed
at UserAuth.test.js:42
$ _
⚡ prtr suggest
→ fix "test 2 failed"
→ debug 분석
→ 무시

B. 플로팅 사이드패널

터미널 옆에 작은 제안 패널. 클릭해서 선택. (iTerm2/Warp 스타일)

⚡ prtr 방금 전
테스트 2개 실패 감지.
AI에 fix 보낼까?

C. 시스템 알림

macOS/Linux 네이티브 알림. 터미널과 분리된 비침투적 제안.

컨텍스트

prtr watch가 감지하는 이벤트: 테스트 실패, git conflict, 빌드 에러, crash log. 선택한 UX가 모든 이벤트에 동일하게 적용돼.
